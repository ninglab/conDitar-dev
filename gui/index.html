Reference

How to use conDitar

Quick guidance for setting up runs, interpreting jobs, and exporting useful subsets of your results.

Set up a run

Choose where to run, then choose Protein + reference ligand or Pocket only. Upload a PDB; reference mode also needs an SDF. Use the folder picker for batches.

Batch behavior

Each folder is processed as its own job; inputs are never mixed. CPU batches run one at a time. Slurm GPU batches can run in parallel when scheduler capacity is available.

Job states

Queued/PENDING means the queue has not started the job. Running means container execution has begun. Failed and canceled jobs include a reason and log paths.

Vina and QVina

The primary Vina score is a standard energy score: lower is better. Vina minimize and dock are separate outputs. QVina is reported separately when selected.

Filter and export

Change a distribution threshold to select candidates automatically, then adjust individual checkboxes in the table. Export the selected subset or uncheck the scope option to export all candidates.

Where files go

Uploaded inputs, outputs, logs, and metadata are stored under job_data/jobs/<job-id>/. Browser downloads go to Downloads; completed-job ZIPs are also saved in the job output folder.

Troubleshooting

For a failure, open the selected job and read the reported log paths. Useful files include job.json, logs/stdout.log, logs/stderr.log, Slurm submission logs, and outputs/.

Pending GPU jobs

A valid Slurm job can remain pending while waiting for account, partition, or GPU capacity. This is a scheduler state, not evidence that the molecule input or code failed.
